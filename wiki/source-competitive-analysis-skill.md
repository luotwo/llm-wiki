# 我用这个 Skill 把竞品分析从 2 天压到 2 小时

## 来源信息

- **作者**：SinKris（公众号"Kris产品成长之路"）
- **发布日期**：2026-03-30

## 核心问题

竞品分析慢的四个根因：
1. **信息收集靠手动** — 打开网页、复制粘贴、截图，每个竞品重复 N 遍
2. **格式每次不统一** — 表格、列表、各种样式混用，无法横向比较
3. **数据容易过期** — 整理完定价就过时了，继续用旧数据做判断
4. **分析结论靠感觉** — "竞品功能较为完善，建议关注"式废话

## 解决方案：两个 Skill 组合

| Skill | 职责 |
|-------|------|
| [[agent-browser]] | 实时打开竞品官网/App Store，抓取当天最新数据 |
| [[competitive-profile-builder]] | 按固定框架结构化分析，保证每次格式一致 |

### 关键优势：数据新鲜度

> 过去用 ChatGPT 或 Deep Research 查竞品定价，得到的经常是几个月前的旧数据。`agent-browser` 是真正打开浏览器访问页面，拿到的是**今天**的价格、功能、版本更新。

## 完整流程（三步走）

### Step 1：建立竞品清单（5 分钟）
创建 `competitive/competitors.md`，列出每个竞品的：
- 官网、定价页、App Store 链接
- 本次重点关注维度
- 此文件长期维护，下次只需改"重点关注"方向

### Step 2：AI 实时抓取（30-40 分钟）
Claude Code + [[agent-browser]] 逐个访问链接，提取：
- 核心产品定位（价值主张）
- 主要功能模块
- 定价体系（套餐名称、价格、权益）
- 近期更新（最近 3 个版本）
- 用户评价亮点（正面/负面高频关键词）

每个竞品生成独立档案，存入 `competitive/profiles/`

### Step 3：生成对比报告（20 分钟）
输出两份文件：
- `comparison_report.md` — 功能矩阵 + 定价对比 + 差异化 + 市场空白 + 启示
- `executive_summary.md` — 一页纸摘要，可直接带去评审

## 效果对比

| 维度 | 以前 | 现在 |
|------|------|------|
| 单个竞品耗时 | 1-2 小时 | 约 6-8 分钟 |
| 5 个竞品总耗时 | 1-2 天 | 约 1 小时 |
| 格式一致性 | 看心情 | 固定框架，完全一致 |
| 数据新鲜度 | 依赖整理时间 | 当天实时抓取 |
| 可复用性 | 低，下次重建 | 高，竞品清单长期维护 |

## 注意事项

- **反爬限制**：部分网站需登录或有反爬机制，抓不到就手动补充
- **版本管理**：建议带日期命名（如 `competitor_a_2026Q1.md`），积累变化历史
- **判断层仍是人的**：AI 给出的"启示"是基于信息对比的建议，战略判断需要结合用户/市场/团队资源理解

## 关键洞察

> 竞品分析从"一项重型任务"变成了"一个可以随时跑的例行流程"。以前季度做一次（因为太累），现在想做就做。

与 [[source-claude-code-n8n-workflow|Claude Code + n8n 工作流]] 的共同主题：
- **工具做信息层，判断层还是你的** — 与"搭工作流能力被自动化，业务理解不会"一脉相承
- [[Skills]] 文件 + 固定框架 = 可复用、越用越快
- 核心价值在于改变**时间分配比例**：减少收集整理，增加思考判断

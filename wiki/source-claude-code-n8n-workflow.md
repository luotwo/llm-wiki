# Claude Code + n8n = 工作流神器

## 来源信息

- **作者**：富百（公众号"富百说"，专注 AI 工作流与智能体搭建）
- **发布日期**：2026-03-03
- **触发灵感**：Cody Schneider 的推文——一个人用 Claude Code + API + n8n，一天干的活顶500强公司一个团队干一年（33万人阅读，3400+收藏）

## 核心观点

**两层自动化叠加产生指数级效率**：
- 第一层：[[n8n]] 帮你 7x24 小时执行任务（定时触发、自动重试、报错有日志）
- 第二层：[[Claude Code]] + [[n8n MCP Server]] 帮你搭建 n8n 工作流本身

传统方式：打开 n8n → 拖节点 → 配参数 → 报错 → 切到 ChatGPT 问 → 复制答案回来改 → 再测试... 时间一半浪费在来回切换上。

新方式：用自然语言告诉 Claude Code 需求，AI 直接调 n8n API 创建工作流、配节点、测试、自动修复报错。**不需要打开 n8n 界面。**

## 三个关键角色

| 角色 | 说明 |
|------|------|
| [[Claude Code]] | Anthropic 命令行 AI 助手，能写代码、操作文件、调 API |
| [[n8n MCP Server]] | n8n 官方接口协议，让 Claude Code 直接操控 n8n 实例 |
| [[Skills]]（.md 文件） | 给 AI 的"说明书"，沉淀业务逻辑和最佳实践，让工作流质量越来越高 |

## 为什么还需要 n8n（而非纯脚本）

1. **透明性**：脚本是黑盒，n8n 每个节点输入输出一目了然，支持权限管理、版本记录、多人协作
2. **成本**：大量固定流程（抓价格、同步库存、发确认邮件）不需要 AI 动脑子，n8n 直接执行更便宜

> 需要动脑子的交给 AI，不需要动脑子的交给 n8n。各干各擅长的事。

## 实际案例

### 跨境电商产品图生成
- 痛点：每月 200+ SKU 上新，每品需主图/场景图/白底图，传统拍摄 2000 元/套
- 方案：飞书表格读取产品信息 → AI 生图 API → 自动裁剪多站点尺寸 → 回写飞书
- 效果：成本从小几万/月降到 200 元以内，运营自己填表点击出图
- 搭建时间：从 2-3 天缩短到大半天

### 其他场景
- **竞品监控**：抓取 ASIN 价格和评价，变动超阈值发飞书通知
- **客户开发**：抓取展会参与者 → 查邮箱 → 验证 → 自动发开发信
- **内容生产**：批量生成 Listing 文案、公众号封面、朋友圈配图

## 关键洞察

> "如果你不理解自己要自动化的流程，你只是在规模化混乱。"

- **搭工作流的能力正在被自动化，理解业务的能力不会**
- 同样用 Claude Code，懂业务和不懂业务的人搭出来的东西差距巨大
- 工具操作 AI 能学会，业务洞察 AI 学不会
- **工作流思维 > 单点工具**：工具会变，业务理解一直值钱

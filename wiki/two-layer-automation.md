# 两层自动化

一种效率叠加模式，通过将"搭建自动化"本身也自动化，实现指数级效率提升。

## 两层结构

**第一层：n8n 帮你执行任务**
- 搭好工作流后 7x24 小时运行
- 定时触发、自动重试、报错有日志
- 处理固定流程，不需要 AI 动脑子

**第二层：AI 帮你搭建 n8n**
- [[Claude Code]] + [[n8n MCP Server]] 理解需求，直接调 API 创建工作流
- [[Skills]] 文件沉淀业务经验，搭出来的工作流越来越贴合实际

## 效率差距

| 方式 | 一天产出 |
|------|---------|
| 手动拖节点 | 3-5 个工作流（上限） |
| Claude Code + n8n MCP | 几十个工作流 |

## 案例佐证

Cody Schneider 一天的产出：40 个 Facebook 广告、100 个着陆页、3 篇客座博客、4 个播客预约、5 篇帮助文档、2 个视频剪辑、25 条推文排期、2 个引流工具——还剩 4 小时。

## 关键前提

> "如果你不理解自己要自动化的流程，你只是在规模化混乱。"

两层自动化放大的是业务理解，不是工具操作。

# LLM Wiki 使用说明

## 简介

LLM Wiki 是一种用 LLM 构建个人知识库的方法论。与传统 RAG（每次查询从头检索）不同，LLM 会**持续构建并维护一个结构化的 Markdown Wiki**，知识随每次导入和查询不断积累，形成复利效应。

> 类比：Obsidian 是 IDE，LLM 是程序员，Wiki 是代码库。

---

## 核心架构

```
项目根目录/
  raw/            # 原始资料（不可变，LLM 只读）
    assets/       # 图片、附件
  wiki/           # LLM 生成维护的知识库页面
    index.md      # 内容索引目录
    log.md        # 操作日志（按时间追加）
  CLAUDE.md       # Schema 配置文件
```

| 层级 | 说明 | 谁管理 |
|------|------|--------|
| Raw Sources | 原始文章、论文、笔记等 | 你 |
| Wiki | 摘要、实体页、概念页、综合分析 | LLM |
| Schema | 结构约定和工作流定义 | 你 + LLM 共同迭代 |

---

## 三大操作

### 1. Ingest（导入资料）

将新资料放入 `raw/` 目录，然后告诉 LLM 处理：

```
请导入 raw/xxx.md 到知识库
```

LLM 将会：
- 阅读资料，与你讨论要点
- 创建摘要页到 `wiki/`
- 更新 `wiki/index.md` 索引
- 更新相关实体页和概念页（可能涉及 10-15 个文件）
- 在 `wiki/log.md` 追加记录
- 标注与已有知识的矛盾

### 2. Query（查询）

直接向 LLM 提问：

```
根据知识库，对比 X 和 Y 的区别
```

LLM 将会：
- 读索引 -> 定位相关页面 -> 深入阅读 -> 综合回答
- 有价值的回答可回写为新 Wiki 页面

### 3. Lint（维护健康）

定期请求检查：

```
请检查知识库的健康状态
```

LLM 将检查：
- 页面间的矛盾内容
- 被新资料取代的过时信息
- 无入站链接的孤立页面
- 缺少专属页面的重要概念
- 缺失的交叉引用
- 可填补的数据空白

## 快速开始

### 1. 初始化

```
请帮我初始化一个 LLM Wiki 知识库
```

### 2. 添加第一份资料

将文章/笔记放入 `raw/` 目录，然后：

```
请处理 raw/my-first-article.md
```

### 3. 开始查询

```
知识库中关于 XX 话题有哪些要点？
```

## 推荐工具搭配

| 工具 | 用途 |
|------|------|
| **Obsidian** | 浏览 Wiki、查看图谱视图、实时预览 |
| **Obsidian Web Clipper** | 浏览器剪藏文章为 Markdown |
| **Marp** | 从 Wiki 内容生成演示文稿 |
| **Dataview** | 基于 frontmatter 生成动态表格 |
| **Git** | 版本控制，天然拥有历史记录 |
| **qmd** | Wiki 页面搜索引擎（BM25 + 向量混合检索） |

## 适用场景

- **个人成长** — 追踪目标、健康、心理、自我提升
- **研究** — 深入某个主题，持续数周/月，逐步构建全面 Wiki
- **读书** — 按章节记录，构建人物、主题、情节线索页
- **商业/团队** — 从 Slack、会议、项目文档中维护内部 Wiki
- **竞品分析、尽职调查、旅行规划、课程笔记** — 任何需要长期积累的场景

---

## 核心理念

> 维护知识库的繁重工作不是阅读和思考，而是记账——更新交叉引用、保持摘要最新、标注矛盾、维护一致性。人类因维护负担增长快于价值而放弃 Wiki。LLM 不会厌倦，不会忘记更新引用，一次操作可以触及 15 个文件。维护成本趋近于零，Wiki 就能持续运转。
